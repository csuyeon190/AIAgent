종합 권고 사항 (5차 분석)

  1. 고객사례 페이지 공통 템플릿 이슈 재확인 (URL 1~5): 모든 영문 고객사례 페이지에서 상단 3종 이미지(썸네일/PC 헤더/모바일 헤더)의 alt가 빈 값입니다. 이는 한글 사이트와 동일한 패턴으로, CMS 또는 페이지 템플릿 차원에서 일괄 개선이 시급합니다.
  2. Tech Report 페이지의 중복 이미지 이슈 (URL 6, 7): Cloud Storage Orchestration과 Datacenter Security 페이지에서 동일한 썸네일 이미지가 페이지 상단에 두 번 게재되어 있습니다(en_thumb04_1102.png 및 en_thumb10_1102.png). 이는 접근성뿐 아니라 콘텐츠 구조 측면에서도 문제이므로 HTML 마크업에서 중복 이미지를 제거하는 것을 권장합니다.
  3. 다국어 사이트 일관성 이슈:
    • URL 6, 7 (Tech Report): "작성자" (한글)로 표시된 Author 레이블 → "Author"로 변경
    • URL 6 (Storage Orchestration): Figure 3 본문 텍스트 일부가 한글("Cloud 서비스, Application & Storage 연결하는 기술")로 표기 → 영어 번역 필요
    • URL 9 (Sustainability Report 2024): 이미지 파일명이 첨부2_372x210.png 등 한글 → 영문 파일명 권장
  4. 인포그래픽/도식 alt 누락 심각 (URL 4, 6, 7, 9): 본문에서 핵심 시각 정보를 전달하는 도식 이미지(에너지 관리 시스템 도식, Storage Orchestration 아키텍처, 네트워크 분리 개념도, ESG 로드맵, Climate Report 등)에 alt가 모두 비어 있습니다. 이러한 이미지는 본문 텍스트만으로는 정보 전달이 충분하지 않으므로 핵심 내용을 요약한 alt 부여가 반드시 필요합니다.
  5. ESG 페이지(URL 8, 9)의 alt 부분적 작성: Sustainability Report 2025 페이지(URL 8)는 이미지 2개에 ESG_250805_01 image 같은 식별자성 alt가 있는데, 이는 의미 없는 파일명 형식 alt이므로 콘텐츠를 설명하는 의미 있는 alt로 변경이 필요합니다. Report 2024 페이지(URL 9)는 모든 본문 이미지의 alt가 비어있어 더 큰 개선이 필요합니다.
  6. "Like" 하트 아이콘 처리: ico_post_heart.png는 단순 좋아요 표시 장식 아이콘이므로 alt="" + aria-hidden="true"로 처리하거나, 의미 있는 경우 "Like" 그대로 유지하는 것을 권장합니다.
  7. 콘텐츠 무결성 이슈 (alt 외 발견사항):
    • URL 1, 3, 5: 인용문에 큰따옴표가 이중으로 표시("...")되어 있는 마크업 오류 - HTML 인코딩 문제로 보이며 별도 수정 권장
    • URL 4 (EHS): 본문 텍스트 일부 오타 ("Intergrated", "Deveop") → "Integrated", "Develop"으로 수정
    이는 alt 문제는 아니지만 사이트 품질 향상을 위해 함께 검토 권장.
  8. 단축형 고객사례 페이지(URL 1, 3)의 본문 콘텐츠 부재: 두 페이지 모두 noindex, nofollow 메타가 설정되어 있고 본문 콘텐츠가 거의 없습니다. 운영자 측에서 실제 페이지 노출 상태를 별도 점검할 필요가 있습니다.
