1. 사용자에게 보이지 않는 트래킹 픽셀(Facebook Pixel, LinkedIn Insight Tag, GTM 등 1x1 픽셀의 noscript용 이미지)은 분석에서 제외하였습니다.
2. 페이지 하단의 공유(Share) 아이콘 SVG, "관련 솔루션" 영역의 작은 솔루션 썸네일 등 페이지마다 동일한 UI 컴포넌트 패턴은 1차 보고서의 공통 권고를 그대로 적용하며 본 보고서에서는 별도 다루지 않습니다.

종합 권고 사항 (2차 분석)

  1. 고객사례 페이지의 공통 패턴 - 상단 3종 이미지 alt 누락: 거의 모든 고객사례 페이지에서 상단의 썸네일.jpg / 피쳐드(PC).jpg / 모바일.jpg 3종 이미지에 alt 속성이 빈 값으로 되어 있습니다. 이는 사이트의 템플릿 수준 이슈로 보이므로 CMS 또는 페이지 템플릿 차원에서 일괄 개선이 필요합니다.
  2. 아키텍처 다이어그램의 정보 손실이 심각: 아모레퍼시픽(amore_str_1.png), 국도화학(bw_arch_5.JPG), Brightics AI(BrighticsAI__1.png) 등의 클라우드 아키텍처 도식은 본문에서 중요한 시각적 정보를 전달함에도 alt 값이 비어있어, 스크린리더 사용자는 본문 텍스트로 도식을 추가 설명하는 부분에 전적으로 의존해야 합니다. 핵심 컴포넌트와 연결 구조를 요약한 alt 추가가 필요합니다.
  3. 이미지가 곧 제목인 사례(코레일 페이지): brityrpa_kospo_title1/2/3.jpg는 "Business Challenges", "Transformation", "Results" 등 섹션 제목을 이미지로 표현한 사례입니다. 일부에는 alt가 있으나("Business Challenges") 일부는 빈 값이며, 한글 사이트이므로 한글 의미를 보강한 alt(예: "Transformation(변화) 섹션 제목")이 더 유용합니다. 가장 좋은 방법은 텍스트로 대체하고 이미지를 장식용으로 처리하는 것입니다.
  4. 화이트페이퍼 페이지의 표지 이미지: 백서 표지는 백서의 제목과 시리즈 정보(WP No.10 등)를 담고 있으므로, "백서 표지" 이상의 의미 있는 정보(제목, 시리즈명, 주제)를 alt에 포함해야 합니다.
  5. 단축형 고객사례 페이지(URL 1, 10): 본 보고서에서 추출된 이미지가 1개로 적습니다. 실 페이지에서 동적 콘텐츠(영상, 동적으로 로딩되는 이미지 갤러리, PDF 임베드 등)가 있을 수 있으므로 운영자가 페이지 내 실제 노출 콘텐츠를 별도 점검하는 것을 권장합니다.
  6. 이전(1차) 보고서의 공통 권고 항목은 본 페이지들에도 동일 적용: 트래킹 픽셀 처리, 공유 아이콘 alt 개선("LinkedIn으로 공유하기" 등), 페이지 하단 "관련 솔루션" 영역의 작은 썸네일 이미지에 대한 alt 개선도 마찬가지로 진행하시면 됩니다.
